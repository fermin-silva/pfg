Instrucciones

Descargar, Compilar, Ejecutar

Introducción

La aplicación permite ejecutar un servicio Web para interactuar con el Algoritmo Genético, así como para recopilar información general, de manera que sea totalmente autocontenida. De esta forma se permite desacoplar el algoritmo de la interfaz gráfica.

De no querer instalar la aplicación, puede accederse a una versión pública online desde la plataforma GitHub pinchando aquí.
Alternativamente, puede ejecutarse la aplicación en el propio ordenador y acceder al navegador. Se verá a continuación como realizarlo.

También se muestran las distintas formas de ejecución e instrucciones sobre cada una.

Instalación Básica

La manera más sencilla de instalar la aplicación es descargando el empaquetado desde el siguiente enlace.
Al no requerir compilación, solo es necesario el entorno de ejecución Java JRE (no JDK). El mismo puede obtenerse desde la página de Oracle.

Una vez descomprimidos los contenidos, se tendrá la siguiente estructura de archivos:

  • lib/: librerias necesarias para ejecutar la aplicación
  • web/: contenidos estaticos para la interfaz web (html, css, etc.)
  • pfg.jar: la aplicación Java ya compilada

La aplicación ya estará instalada y lista para ejecutarse.

Instalación desde el código Fuente

Recordar que para compilar una aplicación Java hace falta de tener instalado el entorno de desarrollo JDK, versión 6 o superior. Descargar desde la página de Oracle.

Para compilar el proyecto utilizaremos la línea de comandos. Se recomienda utilizar Linux o MAC OSX, o en Windows alguna versión de Cygwin.

Pasos

  1. Descargar una copia de la última versión del código: 

    $ git clone https://github.com/fermin-silva/pfg.git

    De no disponer de git, Descargar la versión .zip, luego descomprimirla en una carpeta llamada pfg

  2. Entrar en la carpeta pfg y compilar:

    • En linux o MAC OSX:

      $ sh ./compilar.sh

    • En Windows:

      $ compilar.bat

    3. Esto generará un archivo llamado pfg.jar, listo para ser ejecutado.

Ejecución

Luego de instalar la aplicación, es necesario abrir la línea de comandos y entrar al directorio.
Una vez allí, podremos ejecutar el algoritmo de tres formas distintas:

  1. Por línea de comandos: ejecución básica del algoritmo. Se eligen los parámetros y se imprime el resultado por pantalla en modo texto.
  2. Via Web/HTTP: ejecuta un servidor Web que permite visualizar estas pantallas así como ejecutar el algoritmo eligiendo los parámetros en la interfaz Web.
  3. Pruebas empíricas de rendimiento: ejecuta una batería de pruebas con las distintas recombinaciones implementadas por línea de comandos.

Ejecución por línea de comandos

En los ejemplos se mostrarán ejecuciones en un entorno Linux. De ejecutarse en Windows, utilizar el comando ejecutar.bat en vez de ejecutar.sh

Ejecución de Ejemplo:

$ sh ejecutar.sh -f Schwefel -r Simple -s Torneo -d 10 -g 100

Con ello se ejecutará el algoritmo con la Funcion Schwefel de 10 dimensiones, Recombinación Simple y Torneo Binario, por 100 generaciones.
La salida del algoritmo contendrá el mínimo obtenido y sus cordenadas, así como otros datos de interés.

También se generarán dos imágenes con las curvas de progreso. Tomar nota de las direcciones y abrirlas con cualquier explorador de archivos.
Ejemplo de las imágenes de salida: 

... 
Guardando el archivo en /Users/dev/workspace/pfg/./Schwefel(10)_momento.png
Guardando el archivo en /Users/dev/workspace/pfg/./Schwefel(10)_progreso.png

Ejecución del servidor Web

Nuevamente en la línea de comandos, ejecutar:

$ sh ejecutar.sh -puerto 8080

Esto ejecutará el servidor web en el puerto especificado indefinidamente, hasta que se cancele la ejecución con la señal CTRL+C.

Con el servidor corriendo, sólo basta acceder a un navegador de Internet a la dirección http://localhost:8080 (si se utilizó el puerto 8080).
Se verá esta misma página Web, solo que servida desde el ordenador propio en vez desde uno remoto.

Ejecución de las pruebas de rendimiento

Es similar a la ejecución normal por línea de comandos, sólo que no se utiliza el parámetro de Recombinación (ya que se utilizan todas). Debe agregarse el parámetro -benchmark para indicar que se quieren realizar las pruebas de rendimiento.

La salida del algoritmo es una serie de imágenes (curva de progreso, curva de convergencia de la población y tiempos) por cada función y recombinación en el directorio actual.

Ayuda de ejecución

Ante cualquier duda sobre cómo ejecutar el algoritmo o los parámetros disponibles, puede acudirse a la ayuda autocontenida. Siempre y cuando la aplicación se encuentre ya instalada.

Situados en el directorio donde se encuentra el archivo .jar, ejecutar:

  • En Linux o MAC OSX: 
    $ sh ejecutar.sh -h
  • En Windows: 
    $ sh ejecutar.sh -h

La salida serán todos los parámetros disponibles y una breve descripción de los mismos.
Por ejemplo: 

Usage: <main class> [options]
  Options:
    -alpha
       Para las recombinaciones que utilizan un valor alpha para el promedio
       ponderado. En archivo de properties es 'alpha'
    -benchmark
       Si esta presente se activa el elitismo, de lo contrario se asume que no
       se utiliza. En archivo de properties es 'elitismo'
    -beta
       Para las recombinaciones que utilizan un valor beta para el promedio
       ponderado de la segunda cria. En archivo de properties es 'beta'
    -desviacion
       Para la mutacion normal, el valor de la desviacion tipica. En archivo de
       properties es 'desviacion.mutacion'
    -dimension, -d
       La dimension de la funcion (si es que la funcion asi lo requiere). En
       archivo de properties es 'dimension.funcion'
       Default: 2
...
